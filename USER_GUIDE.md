## 1. 系统简介

### 1.1 AgentHub 是什么

AgentHub 是一个 **AI Agent 管理平台**，旨在为企业内部提供统一的 AI 智能体管理、分发和监控能力。通过 AgentHub，用户可以：

- **浏览与使用**：在应用市场中发现和安装各类 AI Agent
- **开发与发布**：开发者可以创建、提交并管理自己的 Agent
- **管理与运维**：管理员可以审批 Agent、管理用户和部门、监控系统运行状态

### 1.2 系统架构

| 组件 | 技术栈 |
|------|--------|
| 前端 | Vue 3 + Ant Design Vue + Vite |
| 后端 | FastAPI + PostgreSQL + Redis |
| 部署 | Docker + Nginx |

### 1.3 访问地址

| 项目 | 地址 |
|------|------|
| **系统入口** | https://agent.suncentgroup.com/ |
| **服务器 IP** | 120.77.223.91 |
| **代码仓库** | https://github.com/lijiangchao0607/agenthub |

### 1.4 登录方式

AgentHub 支持以下登录方式：

| 登录方式 | 说明 | 适用对象 |
|----------|------|----------|
| **钉钉扫码登录** | 使用钉钉 App 扫码，SSO 单点登录 | 所有用户（推荐） |
| **用户名密码登录** | 输入用户名和密码 | 管理员账号（默认 admin/admin123） |

> ⚠️ **提示**：用户名密码登录仅限管理员账号使用，普通用户请使用钉钉扫码登录。

## 3. 用户功能

### 3.1 应用市场

应用市场是发现和使用 Agent 的主要入口。

#### 浏览 Agent

1. 点击顶部导航栏的 **应用市场**
2. 页面默认展示所有已上线的 Agent 列表
3. 每个 Agent 卡片显示以下信息：
   - Agent 名称和图标
   - 简要描述
   - 开发者/所属部门
   - 安装数量/使用次数
   - 状态标签（已上线/已下线）

#### 搜索 Agent

1. 在应用市场页面顶部的搜索框中输入关键词
2. 系统将实时过滤显示匹配的 Agent
3. 支持按 Agent 名称、描述、开发者进行模糊搜索

#### 筛选 Agent

应用市场支持多维度筛选：

| 筛选条件 | 说明 |
|----------|------|
| 分类筛选 | 按业务类别筛选（如办公效率、数据分析、客服支持等） |
| 部门筛选 | 按所属部门筛选 |
| 排序方式 | 按热度、最新发布、评分等排序 |

#### 安装 Agent

1. 在应用市场中找到需要的 Agent
2. 点击 Agent 卡片上的 **安装** 按钮
3. 安装成功后，可在 **我的应用** 中查看和使用

---

### 3.2 我的应用

"我的应用"页面用于管理已安装和已创建的 Agent。

#### 已安装的 Agent

1. 点击顶部导航栏的 **我的应用**
2. **已安装** 标签页展示所有已安装的 Agent
3. 点击 Agent 卡片即可进入 Agent 对话/使用界面
4. 支持以下操作：
   - **打开**：启动 Agent 进行对话
   - **卸载**：移除已安装的 Agent
   - **查看详情**：查看 Agent 详细信息和版本记录

#### 已创建的 Agent（开发者角色）

拥有 **开发者** 及以上角色的用户，可以看到 **已创建** 标签页：

1. 点击 **已创建** 标签
2. 展示当前用户创建的所有 Agent
3. 支持以下操作：
   - **编辑**：修改 Agent 配置和描述
   - **提交审核**：提交 Agent 给管理员审批上线
   - **查看状态**：查看审批进度（待审核/已通过/已拒绝）

---

### 3.3 运行监控

运行监控页面展示系统整体的运行状态和统计数据，所有用户均可查看。

#### 3.3.1 系统概览

- **系统状态**：当前系统是否正常运行（绿色/红色状态灯）
- **在线 Agent 数量**：当前已上线并可用的 Agent 总数
- **今日调用量**：当天所有 Agent 的总调用次数
- **活跃用户数**：当天的活跃用户数量

#### 3.3.2 调用趋势

- **时间范围选择**：支持按今日/本周/本月查看
- **趋势图表**：以折线图/柱状图展示 Agent 调用量随时间的变化
- **指标**：调用次数、成功率、平均响应时间

#### 3.3.3 资源监控

- **CPU 使用率**：服务器 CPU 使用情况
- **内存使用率**：服务器内存占用情况
- **磁盘使用率**：存储空间使用情况
- **网络流量**：入站/出站网络流量

> 💡 **提示**：资源监控数据帮助了解系统负载情况，当资源使用率持续偏高时，建议联系管理员进行扩容。

#### 3.3.4 Agent 排行

| 排行维度 | 说明 |
|----------|------|
| 热度排行 | 按调用次数排序，展示最热门的 Agent |
| 好评排行 | 按用户评分排序 |
| 最新上线 | 按上线时间排序，展示最新上线的 Agent |

#### 3.3.5 错误日志

- **错误列表**：展示最近的系统错误和异常记录
- **错误详情**：点击可查看错误的具体信息，包括时间、Agent、错误类型和堆栈
- **筛选功能**：支持按时间范围、Agent、错误级别筛选

> ⚠️ **注意**：普通用户仅能查看与自己使用的 Agent 相关的错误日志，管理员可查看所有错误日志。

## 4. 管理员功能

> ⚠️ **注意**：管理后台功能仅对拥有管理员角色（super_admin、agent_admin、dept_admin）的用户可见。如看不到管理后台入口，请联系超级管理员分配权限。

进入管理后台：点击顶部导航栏的 **⚙️ 管理后台**

### 4.1 管理概览

管理概览是管理员进入后台后的默认页面，提供全局统计数据和快捷操作。

#### 统计卡片

| 统计项 | 说明 |
|--------|------|
| Agent 总数 | 系统中所有 Agent 的数量（含上线和下线） |
| 已上线 Agent | 当前已上线可用的 Agent 数量 |
| 注册用户总数 | 系统中的总用户数 |
| 今日活跃用户 | 当天登录并使用过系统的用户数 |
| 今日总调用量 | 当天所有 Agent 的总调用次数 |
| 待审批 Agent | 等待管理员审批的 Agent 数量 |

#### 快捷操作

快捷操作面板提供常用管理功能的快速入口：

| 快捷操作 | 说明 |
|----------|------|

| 审批 Agent | 跳转到待审批列表 |

| 查看审计日志 | 跳转到审计日志页面 |

### 4.2 Agent 管理

Agent 管理是管理员的核心功能之一，支持 Agent 的全生命周期管理。

#### 4.2.1 创建 Agent

1. 在管理后台左侧菜单中点击 **Agent 管理**
2. 点击 **创建 Agent** 按钮
3. 填写 Agent 基本信息：
   - **名称**：Agent 的显示名称（唯一）
   - **描述**：Agent 的功能说明
   - **分类**：选择业务类别
   - **图标**：上传 Agent 图标
   - **版本号**：初始版本号（如 v1.0.0）
   - **API 配置**：Agent 对接的 API 地址和参数
   - **模型配置**：选择使用的 AI 模型和参数
4. 点击 **保存** 完成创建

#### 4.2.2 审批 Agent

开发者提交 Agent 后，需要管理员审批才能上线：

1. 进入 **Agent 管理** 页面
2. 切换到 **待审批** 标签
3. 查看待审批的 Agent 列表
4. 点击某个 Agent 的 **审批** 按钮
5. 查看 Agent 详情，做出审批决定：
   - **通过**：Agent 状态变为已上线，用户可以在应用市场中看到并安装
   - **拒绝**：填写拒绝原因，Agent 退回给开发者修改

#### 4.2.3 部署 Agent

1. 在 Agent 管理列表中找到目标 Agent
2. 点击 **部署** 按钮
3. 选择部署环境（测试/生产）
4. 系统自动执行部署流程，页面显示部署进度
5. 部署完成后，Agent 状态更新

#### 4.2.4 Agent 上线/下线

- **上线**：将已审批通过的 Agent 发布到应用市场，用户可以搜索和安装
  - 在 Agent 列表中，找到已审批但未上线的 Agent
  - 点击 **上线** 按钮，确认操作
  
- **下线**：将已上线的 Agent 从应用市场移除
  - 在 Agent 列表中，找到已上线的 Agent
  - 点击 **下线** 按钮，确认操作
  - ⚠️ **注意**：下线后，已安装该 Agent 的用户将无法继续使用

### 4.3 用户管理

#### 4.3.1 查看用户列表

1. 在管理后台左侧菜单中点击 **用户管理**
2. 页面展示所有用户列表，包含以下信息：
   - 用户名/姓名
   - 所属部门
   - 角色
   - 状态（正常/锁定）
   - 最后登录时间

#### 4.3.2 添加用户

1. 在用户管理页面点击 **添加用户** 按钮
2. 填写用户信息：
   - **用户名**：登录账号
   - **姓名**：真实姓名
   - **邮箱**：工作邮箱
   - **手机号**：联系电话
   - **所属部门**：选择部门
   - **角色**：分配用户角色
   - **初始密码**：设置初始登录密码
3. 点击 **保存** 完成创建

#### 4.3.3 编辑用户

1. 在用户列表中找到目标用户
2. 点击 **编辑** 按钮
3. 修改需要更新的信息（姓名、部门、角色等）
4. 点击 **保存**

#### 4.3.4 锁定/解锁用户

- **锁定用户**：禁止用户登录系统
  - 在用户列表中点击 **锁定** 按钮
  - 锁定后用户无法登录，但数据保留
  
- **解锁用户**：恢复用户的登录权限
  - 在用户列表中点击 **解锁** 按钮

#### 4.3.5 重置密码

1. 在用户列表中找到目标用户
2. 点击 **重置密码** 按钮
3. 系统生成新的临时密码或手动输入新密码
4. 将新密码通过安全渠道告知用户
5. 建议用户首次登录后立即修改密码

### 4.4 部门管理

#### 4.4.1 部门树

1. 在管理后台左侧菜单中点击 **部门管理**
2. 页面左侧展示部门树形结构
3. 支持展开/折叠查看下级部门

#### 4.4.2 创建部门

1. 在部门树中，右键点击父部门节点
2. 选择 **添加子部门**
3. 填写部门信息：
   - **部门名称**：部门全称
   - **部门编码**：部门唯一标识
   - **上级部门**：选择所属的上级部门
   - **负责人**：指定部门负责人
4. 点击 **保存**

#### 4.4.3 编辑部门

1. 在部门树中点击目标部门
2. 在右侧详情面板中点击 **编辑**
3. 修改部门信息
4. 点击 **保存**

#### 4.4.4 添加/移除成员

**添加成员：**

1. 点击目标部门
2. 在右侧成员列表中点击 **添加成员**
3. 在弹出的用户选择器中搜索并勾选用户
4. 点击 **确认** 完成添加

**移除成员：**

1. 点击目标部门
2. 在成员列表中找到目标用户
3. 点击 **移除** 按钮
4. 确认操作

## 5. 角色与权限说明

### 5.1 角色详细说明

AgentHub 系统预置了 5 种角色，满足不同层级的管理需求：

#### ① super_admin（超级管理员）

- **定位**：系统最高权限管理者
- **权限范围**：拥有所有模块的完整操作权限
- **典型职责**：
  - 系统整体管理和配置
  - 分配其他管理员角色
  - 管理所有部门、用户和角色
  - 审批和部署所有 Agent
  - 查看所有审计日志
- **适用人员**：IT 部门负责人、系统运维负责人

#### ② agent_admin（Agent 管理员）

- **定位**：Agent 生命周期管理者
- **权限范围**：
  - ✅ 管理 Agent 的审批、部署、上线、下线
  - ✅ 查看运行监控数据
  - ✅ 查看审计日志（Agent 相关）
  - ❌ 无法管理用户、部门、角色
- **典型职责**：
  - 审批开发者提交的 Agent
  - 管理 Agent 的部署和版本
  - 监控 Agent 运行状态
- **适用人员**：AI 平台运营人员、技术负责人

#### ③ dept_admin（部门管理员）

- **定位**：部门级管理者
- **权限范围**：
  - ✅ 管理本部门用户（添加、编辑、锁定）
  - ✅ 查看本部门的运行监控数据
  - ✅ 管理本部门的 Agent
  - ❌ 无法跨部门管理
  - ❌ 无法修改系统角色权限
- **典型职责**：
  - 管理部门成员
  - 审批部门内的 Agent
  - 查看部门使用情况
- **适用人员**：各部门 IT 联络人、部门经理

#### ④ developer（开发者）

- **定位**：Agent 创建者
- **权限范围**：
  - ✅ 创建和编辑自己的 Agent
  - ✅ 提交 Agent 审批
  - ✅ 查看自己 Agent 的运行数据
  - ✅ 使用应用市场中所有已上线的 Agent
  - ❌ 无法审批或部署 Agent
  - ❌ 无法管理用户和部门
- **典型职责**：
  - 开发和维护 AI Agent
  - 提交 Agent 上线申请
  - 处理 Agent 运行问题
- **适用人员**：AI 开发工程师、算法工程师

#### ⑤ user（普通用户）

  - ✅ 浏览应用市场
  - ✅ 安装和使用已上线的 Agent

  - ❌ 无法创建 Agent
  - ❌ 无法访问管理后台
- **典型职责**：
  - 使用 Agent 完成日常工作任务
  - 反馈 Agent 使用问题和建议

## 6. 常见问题 FAQ

### Q1：登录失败怎么办？

**症状**：钉钉扫码后无法登录，或提示登录失败。

**排查步骤**：

1. **检查网络连接**：确保可以正常访问 `https://agent.suncentgroup.com/`
2. **确认钉钉账号**：确保使用的是公司钉钉账号扫码，非个人钉钉
3. **检查账号状态**：联系管理员确认您的账号是否已被创建且未被锁定
4. **检查角色权限**：确认管理员已在系统中为您创建了账号并分配了角色
5. **清除浏览器缓存**：尝试使用无痕/隐私模式打开页面重新登录
6. **更换浏览器**：推荐使用 Chrome、Edge 等现代浏览器

> 如以上步骤均无法解决，请联系 IT 支持或系统管理员。

---

### Q2：看不到管理后台怎么办？

**症状**：登录后顶部导航栏中没有 **管理后台** 入口。

**原因分析**：

管理后台仅对管理员角色可见。您当前的角色可能是 **developer** 或 **user**，没有管理后台的访问权限。

**解决方法**：

1. 联系您的上级或部门管理员
2. 请管理员按照以下步骤为您分配角色：
   - 进入 **管理后台 → 用户管理**
   - 找到您的账号，点击 **编辑**
   - 将角色修改为需要的管理员角色（如 dept_admin）
   - 点击 **保存**
3. 重新登录系统即可看到管理后台

> ⚠️ **注意**：角色分配需要 super_admin 或 dept_admin 权限。请合理申请所需角色，遵循最小权限原则。

---

### Q3：如何给用户分配管理员角色？

**前提条件**：操作者需拥有 super_admin 角色。

**操作步骤**：

1. 使用 super_admin 账号登录系统
2. 进入 **管理后台 → 用户管理**
3. 在用户列表中找到目标用户
4. 点击该用户的 **编辑** 按钮
5. 在角色下拉选择框中选择目标角色：
   - `agent_admin` — Agent 管理员
   - `dept_admin` — 部门管理员
   - `super_admin` — 超级管理员（谨慎分配）
6. 点击 **保存** 完成角色分配
7. 通知用户重新登录以使新角色生效

**角色分配建议**：

| 场景 | 推荐角色 |
|------|----------|
| AI 平台运营人员 | agent_admin |
| 部门 IT 联络人 | dept_admin |
| Agent 开发人员 | developer |
| 普通业务人员 | user |
| 系统最高管理者 | super_admin |

> ⚠️ **安全提示**：
> - super_admin 角色应严格控制，建议不超过 2-3 人
> - 定期审查角色分配，及时回收离职或转岗人员的权限
> - 所有角色变更操作都会记录在审计日志中

### Q4：如何修改管理员登录密码？

**操作步骤**：

1. 登录系统后，点击页面右上角的用户头像/姓名
2. 选择 **个人设置** 或 **修改密码**
3. 输入当前密码和新密码
4. 点击 **确认修改**

> 如果忘记当前密码，请联系超级管理员通过 **用户管理 → 重置密码** 进行重置。
